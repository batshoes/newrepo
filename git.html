GIT BASICS

Git is version control software that is built into your computer. It's used to track the alterations you make to files in a way that dynamically changes the way software is developed.

git init

Initializes a git repositry on your local machine. Creates a .git_ignore file and begins tracking all other files in the root directory not specified in the ignore file.

git status

Relays information on files being tracked or otherwise. Changing the contents of a file, adding or deleting a file, any of these will cause a file to become "untracked" and ready to start being watched.

git add filename

Adding a file that was previously "untracked" to be committed will move it into the "green zone" and the changes made will be recorded in the next commit. Two ways of adding are:

  • Specific filename
  • " . " - or by root directory

git commit

Commiting files records the changes you have designated in the commit log. These changes are retrievable via the commit number.

  • git commit - opens VIM and asks for a commit message to describe in detail the changes made.
  • git commit -m "short commit message" - commits with a shorter, more poigent message.

GITHUB

Github is an online resource used by developers worldwide to host git repositories. It has become a haven for open source projects, allowing fluid collaboration with multiple developers and companies, encouraging the collective acceleration of learning.

Github Repo

Creating an online repositry on github gives developers the chance to host their own work on github. Once the repository has been created, several git commands will become available.

git push

Sends information on the local machine to the github remote storage.

git pull

When information is altered on Github, but not on the local machine, it is neccesary to "pull" the information from the remote storage.

git checkout branch name

When working on git repositories and implementing new features, it is sometimes wise to make those changes and implementations on a clone of the repository, make sure they work, and then "merge" them into the master branch.
